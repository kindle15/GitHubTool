# Changelog

All notable changes to GithubTool will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),

## [2.0.2] - 2026-03-12

### 🎉 Release - "Console-First / Smart Actions"

This version overhauls the right panel with a tabbed layout and introduces
new repo management actions that all output to the diagnostics console rather
than spawning popup dialogs.

### Added

#### Tabbed Action Panel
- **⭐ Actions tab** - Open in Browser, Clone, Download ZIP, Star/Unstar
- **🔧 Manage tab** - Toggle Privacy, Fork Repo, Rename Repo
- **⚠️ Danger tab** - Delete Repository
- All three tabs share the same right-panel space (icon-only tab strip)

#### Toolbar Buttons
- **🔑 Token button** - Opens GitHub token creation page when no token present;
  shows masked token info in console when token is already set
- **📊 Stats button** - Dumps per-tab repository statistics to the diagnostics
  console: repo counts, privacy breakdown, stars/forks totals, top languages,
  largest repo — no popup dialogs

#### Manage Actions
- **Toggle Privacy** (`_toggle_privacy_async`) - Switch a repo Public ↔ Private
- **Fork Repo** (`_fork_repo_async`) - Fork a selected repo to your account
- **Rename Repo** (`_rename_repo_async`) - Rename a repo on GitHub

#### Console-First Design
- All action results (fork, rename, toggle privacy) written to diagnostics console
- Confirmations via dialog; results via console only

### Changed
- Right panel "⭐ Actions" section replaced with tabbed notebook (⭐/🔧/⚠️)
- `APP_TITLE` updated to `"GithubTool v2.0.2"`
- Creation Lab branding label updated to `v2.0.2`
- All `User-Agent` headers updated to `GithubTool/2.0.2`

### Fixed
- None (new features only)

### Security
- None

## [2.0.1] - 2026-03-10

### 🎉 Release - "Branch Out"

This version adds the Push to Branch workflow, a Publish Existing Folder
shortcut, repository deletion, and fixes the ZIP download.

### Added

#### Push to Branch
- **`_push_to_branch_async()` / `_do_push_to_branch()`** - Push an existing
  local git repo to a named branch on an existing GitHub remote
- **Push to Branch UI section** in Tab 5 (Creation Lab): folder picker, repo
  owner/name fields, branch name field, commit message field
- **🔍 Check Status button** - Checks git status before pushing
- **⬆️ Push to Branch button** - Executes the push

#### Publish Existing Folder
- **`_publish_existing_folder_async()` / `_do_publish_existing_folder()`** -
  Full workflow: git init → create GitHub repo → configure remote → commit all
  files → push
- **🚀 Publish Folder button** in Creation Lab
- **Rollback on failure** - Deletes the newly created GitHub repo if push fails
- **Repo exists handling** - Offers to push to existing repo if name is taken

#### Danger Tab / Delete Repo
- **`_delete_github_repo()`** helper - Delete a GitHub repo via API
- **Delete confirmation dialog** - Requires typing the repo name to confirm

### Changed
- Creation Lab branding label updated to `v2.0.1`
- Added copyright notice in Tab 5 footer

### Fixed
- **Download ZIP** - Fixed ZIP download in `ExternalDownloadTab.download_source()`
  (`User-Agent` updated to `GithubTool/2.0.1`)

### Security
- Rollback prevents orphaned GitHub repos on push failure

## [2.0.0] - 2026-02-16

### 🎉 Major Release - "Safety First"

This major version introduces comprehensive safety checks to prevent accidental data loss when publishing repositories to GitHub.

### Added

#### Safety Features
- **Pre-Publish Repository Checks** - Verifies if repository already exists on GitHub before attempting to create it
- **`check_if_repo_exists()` function** - New API function to check repository existence
- **`get_authenticated_user()` function** - Retrieves username of authenticated user
- **Safety Warning Dialog** - Shows detailed repository information when attempting to publish to existing repo:
  - Repository full name
  - Last updated date
  - Last pushed date
  - Size in KB
  - Default branch name
  - GitHub URL
- **Safe Action Options** in warning dialog:
  - **Cancel (Safest)** - Abort publishing operation
  - **Add as Remote Only** - Link to existing repo without creating new one
  - **View on GitHub** - Open existing repo in browser for inspection

#### Copyright & Documentation
- **Copyright Notice** - Added `(c) kindle15 / 2026` to About dialog
- **Copyright in File Header** - Added to all source files
- **Enhanced Welcome Tab** - Updated with safety features documentation
- **Safety Check Documentation** - Added to DEVELOPER.md and README.md

#### Diagnostics & Logging
- **Safety Check Logging** - Detailed diagnostics messages for:
  - Repository existence checks
  - User authentication
  - Warning dialog interactions
  - Safe action selections
- **Enhanced Startup Messages** - Clear indication that safety checks are enabled

### Changed

#### Publishing Workflow
- **`publish_to_github()` method** - Now includes pre-publish safety checks
- **Creation Lab Publishing** - Enhanced with existence verification before creating repo
- **Error Handling** - Improved error messages for repository conflicts

#### User Experience
- **Warning Dialogs** - More informative with actionable options
- **Button States** - Publish button only enabled after local repo creation
- **Diagnostics Messages** - Clearer indicators for safety check results

### Fixed
- None (new features only, no bug fixes)

### Security
- **Prevents Data Loss** - No longer possible to accidentally overwrite existing repositories
- **User Confirmation** - Explicit user choice required when repo name conflicts detected
- **Safe Defaults** - Cancel option is most prominent in warning dialogs

## Version Numbering

**Format:** MAJOR.MINOR.PATCH

- **MAJOR** - Incompatible API changes or major rewrites
- **MINOR** - New features, backward compatible
- **PATCH** - Bug fixes, backward compatible

**Drop Notation:**
- "Drop X.Y" refers to development iterations
- Drops were consolidated into version numbers for release

**Mapping:**
- v2.0.2 → Tabbed action panel, Token/Stats buttons, Toggle Privacy, Fork, Rename
- v2.0.1 → Push to Branch, Publish Existing Folder, Delete repo, ZIP fix
- Drop 3.8 → v2.0.0 (Safety checks = major feature)

- Drop 3.6 → v1.6.0 (Creation Lab)
- Drop 3.5 → v1.5.0 (Unified layout)

## Upgrade Guide

### From v2.0.x to v2.0.2

**Breaking Changes:** None

**New Features:**
- Tabbed right panel (⭐ Actions / 🔧 Manage / ⚠️ Danger) replaces old single section
- Token button in toolbar
- Stats button in toolbar
- Toggle Privacy, Fork, Rename actions in Manage tab
- Push to Branch and Publish Existing Folder in Tab 5 (added in v2.0.1)

**Action Required:** None — update `gui.py` and run

**Config Migration:** Automatic (no new keys)

**Token Setup:** If you were following old docs that said to save `.github_token`
or `~/.github_token`, those files will NOT be detected. Create
`PersonalAccessToken.txt` on any drive D:–Z: instead, or set the `GITHUB_TOKEN`
environment variable.

---

### From v1.x to v2.0.0

**Breaking Changes:** None

**New Features:**
- Safety checks before publishing (automatic)
- Enhanced warning dialogs (shown when needed)
- Copyright notice in About dialog

**Action Required:** None - update and run

**Config Migration:** Automatic (no changes needed)

**Testing Recommended:**
1. Create new local repo in Creation Lab
2. Try publishing to existing repo name
3. Verify safety dialog appears
4. Test all three options (Cancel, Add Remote, View)

---

### From v0.x to v1.x

**Breaking Changes:** None

**New Features:**
- Creation Lab (Tab 5)
- Star/Unstar functionality
- Export to CSV/TXT
- README viewer
- Resizable diagnostics

**Action Required:** None - update and run

**Config Migration:** New keys added automatically:
- `export_format` (default: "txt")
